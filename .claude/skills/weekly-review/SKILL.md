# weekly-review

Once a week, the board gets swept and the week gets named. This is how stale rows die and how next week starts with intent instead of residue.

## The sweep

1. Read every row of `status-board.md`, including P3.
2. For each row: did it move this week? If resolved, move it to Archived with a one-line resolution note. If untouched for two weeks, ask whether it is still real or should be archived as stale.
3. Read this week's `episodes/` files for anything that happened but never landed on the board.

## The review shape

One message:

- **Moved**: what actually progressed this week, one line each.
- **Stalled**: what did not, and the honest reason if you know it.
- **Decisions made**: anything they decided this week worth remembering. Confirm each is captured in `knowledge/`.
- **Next week**: the 1 to 3 things that matter most, proposed by you, confirmed by them.

After they confirm next week's priorities, update the status board so Monday's daily brief opens on them.

## Tone

This is a working review, not a performance review. No cheerleading, no "great week!". Name things plainly. If the same item stalls three weeks running, say so; that is the most useful sentence in the review.
